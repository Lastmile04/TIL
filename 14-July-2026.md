# What I Learned

### GO-Lang learning 

- I learned GO syntax for things like structs and what actaully interfaces in GO are.
    - structs are similar to how C handles them
    - interfaces are a way to generalize a method or it's just technically a type that defines a set of method signatures.
        - helps in polymorphism by taking in any type as long as the contract set is satisfied. eg. multiple structs having similar method in interface now a usage of structs can be generalized if they have the same method as an interface
        - can improve testability as they can be to implement mock database 
        - makes the codebase much easier to manage and understand 
- How to initialize a Map in GO using `make`
- What Runes actaully are in GO and how to use them
    - basically 'rune' are unicode point numbers which represent a char
    - good to use on strings, the differece there is between string defined as an array of `rune` instead of a simple string is that in a normal string char is of uint8 while in a `rune` array a `rune` is of uint32.
- Goroutines, what they actually are and some functions related to them 
    - A way to launch multiple functions and have them execute concurrently 
    - uses the `go` keyword in front of the function you want to execute concurrently this basically make the program stop waiting for the function to execute 
    - uses `WaitGroups` imported through `sync` package 
        - `WaitGroups` are simply counter which we increment by one whenever we spawn a new goroutine using `.Add(1)` method
        - Whenever a routine finishes executing we use the `.Done()` method to decrement the counter
        - `.Wait()` method is used to wait for the counter to be 0 meaing all tasks have been completed and then the rest of the code will execute
    - if data is being modified at the same time by multiple goroutines collision can happen which can cause corroupted memory and other issues, to tackle this GO uses `Mutex`
        - From the `sync` package `Mutex` short for Mutual exclusion gives use methods like `.Lock()` and the `.Unlock()`
        - we can place these `.Lock()` in the line above where we write data and below that line we place `.Unlock()` 
        - what happens is that when a goroutine reaches this lock method a check is performed to see if a lock is already been set by another goroutine
        - if it is set the routine will wait there untill the lock has been released and then set the lock itself
        - that's why the `.Lock()` & `.Unlock()` methods has to be on above and below the data write line
- I learnt pointers in Goroutines which are similar to how they're handled in C so there was not much to see 
    - There are some differece like in GO pointer arithmetic is not allowed unlike in C where it is allowed and heavily used

### Some networking concepts revision 

- networking concepts like what is a socket, types of sockets 
- commonly used sockets `SOCK_STREAM` and `SOCK_DGRAM`
    - `SOCK_STREAM` : TCP is the protocol used on this socket 
    - `SOCK_DGRAM` : UDP is the protocol used on this socket 
- already knew most of the information but was a good revision 
- Data Encapsulation: a packet is born, the packet is wrapped (“encapsulated”) in a header (and rarely a footer) by
the first protocol (say, the TFTP protocol), then the whole thing (TFTP header included) is encapsulated
again by the next protocol (say, UDP), then again by the next (IP), then again by the final protocol on
the hardware (physical) layer (say, Ethernet).
- When another computer receives the packet, the hardware strips the Ethernet header, the kernel strips the IP and UDP headers, the TFTP program strips the TFTP header, and it finally has the data.
    - `send()`: for stream sockets
    - `sendto()`: for datagram sockets
